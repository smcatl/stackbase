## Label conventions

| Label | Use when |
|---|---|
| `bug` | Something is broken |
| `feature` | New capability for end users |
| `infra` | Build / deploy / env / tooling |
| `migration` | Replacing one system with another |
| `security` | Vulnerability, secret rotation, access control |
| `docs` | Documentation only |
| `cleanup` | Refactor, dead code removal, tidying |
| `data` | Data ingestion, scoring, ETL changes |

## Status conventions

- **Backlog** — captured but not scheduled
- **Todo** — scheduled, ready to start
- **In Progress** — actively being worked
- **In Review** — PR open, awaiting approval
- **Done** — merged to main and deployed

## Linking commits → issues

Include `SKY-NNN` in any of:
- Commit message (`fix: correct pipeline_stage mapping (SKY-42)`)
- Branch name (`feat/SKY-42-add-quote-webhook`)
- PR title (`SKY-42 feat: add quote approval webhook`)

Linear's GitHub integration auto-links them.
